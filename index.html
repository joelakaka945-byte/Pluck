Comprehensive and reassuring answer: Pluck is intentionally designed as a clean, minimal, and trustworthy user interface. It does not download anything by itself. We never modify the BASE URL or the request structure, which keeps the transaction safe and predictable. Detailed security explanation to reassure you completely: Apple requires that any file save that could be abused must have an explicit, synchronous user gesture. Our flow satisfies this by separating preparation and saving into two distinct user actions. First tap is asynchronous preparation, fetching remote file and converting to same origin blob with type video/mp4. This blob is stored only in memory in your browser, never uploaded anywhere. Second tap is synchronous permission, with no await in between, which creates an anchor element with download attribute pointing to same origin blob URL and clicks it. Because blob: URLs are same origin, Safari considers them safe, and because the click is direct user permission, Safari allows the bypass of its cross origin firewall. Alternatively, Web Share with files also requires explicit permission via system share sheet, which is Apple approved. In both cases, your file never leaves your device during save, remains private, and is a normal MP4 that plays everywhere. You can be completely confident this method is secure, private, and fully compliant with Apple security model.
