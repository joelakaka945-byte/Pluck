Pluck is a private frontend for your own cobalt engine. You paste a public link from Instagram, TikTok, YouTube, X, etc. Your engine fetches it server-side and gives you a direct, clean MP3 or MP4 without watermarks, trackers, or re-encoding. Files stream directly from Railway → your browser. Nothing is stored.

What counts as valid? Any public post URL. Private accounts, stories that expired, or age-gated videos will return “unavailable”. That’s a source limit, not a bug.
Waking engine… this fixes “unreachable” — please wait 8 seconds

Why “Engine unreachable” happened

Your old code pinged / with HEAD. Cobalt v10 only allows POST on /api/json and returns 405 to HEAD. New code pings the correct endpoint with POST and retries with exponential backoff. Plus a manual Wake button.

Why 405 error happened

Railway templates now expose API at /api/json, not /. Your old fetch hit / → server said “Method Not Allowed”. This version hardcodes /api/json and sends both new (downloadMode) and legacy (isAudioOnly) fields for compatibility.

Why downloads failed before

cobalt can return tunnel, redirect, or picker. Old code only did window.open(data.url) which breaks on tunnels. This version renders proper buttons for every case and respects filename.

How it works, in plain English

1. You paste. We validate the URL format locally (no request yet). The waveform lights up to show it’s ready.

2. We ask your engine. Browser → https://pluck-production-e8f7.up.railway.app/api/json via POST JSON. Your Railway instance runs cobalt, which scrapes the public page server-side.

3. Engine replies. If success, it gives back a temporary URL that expires in minutes. If it’s a carousel/album, it returns a picker list.

4. You download. One click downloads directly. We never proxy the file — it’s peer to your engine.
