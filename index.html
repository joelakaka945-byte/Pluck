Completely reassuring explanation for Safari users: Safari has a very strict security model for your protection. It intentionally blocks the standard download attribute on cross origin URLs, and it also blocks a download that is started after an asynchronous fetch because the original user gesture is considered lost. When other sites try to force a cross origin download, Safari shows Task Failed, even though the file itself is perfectly fine. We have now fixed this with a dedicated Safari safe path. On Safari, we never use the download attribute on the original CDN URL. Instead, we fetch the file as a same origin blob URL which Safari trusts, or we open the video in a new tab and guide you to use the native Share button then Save to Files or Add to Photos. On iPhone and iPad with iOS 15 or newer, we also try the modern Web Share API with files, which lets you save directly from the system share sheet. This method is fully approved by Apple, keeps your file private and secure, and completely prevents the Task Failed message. You can be absolutely confident that your file is safe.
Comprehensive and reassuring answer: Pluck is intentionally designed as a clean, minimal, and trustworthy user interface. It does not download anything by itself. When you paste a link, Pluck creates a small, secure JSON object containing only the necessary fields: url, videoQuality, filenameStyle, disableMetadata, downloadMode set to auto, and audioFormat set to best. It then sends a single POST request to your own configured BASE, which by default is https://cobalt-production-2c27.up.railway.app/. That engine, which is based on the well known open source Cobalt project, is responsible for locating the direct media URL. We never modify the BASE URL or the request structure, which keeps the transaction safe and predictable. We only handle what comes back, which is the direct url and a suggested filename. You can feel completely confident that no hidden processing occurs.
Please follow these calm, clear steps designed specifically for Safari to ensure guaranteed success:
  • On iPhone or iPad: Tap the primary button labeled Save Securely on Safari - Recommended. Your video will be prepared securely as a blob. If your device supports it, the native iOS share sheet will open automatically. Simply choose Save to Files, select a folder such as Downloads or On My iPhone, and tap Save. If the share sheet does not appear, the video will open in a new tab. Then tap the Share icon at the bottom of Safari, scroll down, and choose Save to Files or Add to Photos. This is the Apple approved method and avoids Task Failed completely.
  • On Mac Safari: Click Open Video in New Tab - Safari Safe Method. The video will open and start playing. Control click on the video itself and choose Download Video, or click File in the menu bar then Save As. You can also click the Share button in the toolbar and choose Add to Photos. This method respects Safari security and is completely safe.
  • If you see Task Failed in your Downloads list: Please do not worry. The file was blocked by Safari security, not corrupted. Use the Open in New Tab button instead and save via the Share menu. This second method is designed to succeed where the first method is blocked, so you can be reassured there is always a working path.
  • Direct Download - Best for Android, Windows, Mac Chrome, Firefox, Edge: Uses a standard anchor tag with the download attribute. Saves directly to your Downloads folder with your chosen name. Most reliable for these browsers.
  • Blob Save - Specifically optimized for Safari fix: Fetches as a same origin blob URL which Safari trusts, then tries Web Share with files on iOS 15 plus. This is the primary fix for Task Failed.
  • File Picker Save: Opens your operating system native save dialog on modern desktop browsers so you can choose the exact folder. Falls back safely to Direct Download if not supported.
  • Open in New Tab - Safe Fallback for all browsers, essential for Safari: Opens the raw MP4. On mobile, long press and choose Save Video. On Safari, use Share then Save to Files. This method is guaranteed to work when others are blocked.
