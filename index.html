Here is the complete, transparent technical explanation to reassure you: First, we attempt a lightweight HEAD request and read the Content-Length header, which is the standard way servers announce file size. If the host blocks HEAD requests or does not allow cross origin reading of that header, we then attempt a GET request with a Range header of bytes 0-0 and read the total size from the Content-Range header, which typically looks like bytes 0-0 / 12345678. If the host also blocks reading CORS headers, which some CDNs do for privacy, we retry the exact same requests through two well known CORS proxies, https://api.allorigins.win/raw and https://corsproxy.io, which add Access-Control-Allow-Origin so we can read the headers safely. This is why you will often see an accurate size like 24.73 MB MP4 instead of unknown. If the host deliberately hides the size even through a proxy, we will honestly display Size hidden by host. In that rare case, the file is still perfectly safe to download, the host simply chose not to reveal the size in advance. This transparency is intentional to keep you fully informed.
