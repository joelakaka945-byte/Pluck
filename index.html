Answer: Pluck is a clean user interface. You paste a link, we create a JSON object containing url, videoQuality, filenameStyle, disableMetadata, downloadMode auto, and audioFormat best, and send a POST request to your base URL, which is https://cobalt-production-2c27.up.railway.app/. The engine then locates the direct MP4 file. We do not modify the base URL or the request structure, so the transaction remains secure. We only process the returned url and filename to display the preview and download options.
Yes, supported platforms include: Instagram Reels, Posts, and Stories if they are public. TikTok standard videos, usually without watermark if the engine provides it. If the account is private, followers only, or requires login, the engine cannot access it. This is a platform restriction, not a bug. Public links provide the best results. The same process applies to YouTube, X, and Reddit.
  • 1 Paste: Copy the full link using the Share button on Instagram or TikTok. The link must start with https://
  • 2 Click Download: The main tool is at the top of the page and is visible immediately without scrolling.
  • 3 View the live log: Displays validation, engine status check, request creation, waiting time, response parsing, filename generation, and size detection.
  • 4 Preview: The video will stream only, no file is saved yet. This allows you to verify the correct video.
  • 5 Select the download method for your device: Please refer to the next question to choose the appropriate button.
  • 6 If an issue occurs: Click Refresh Page to Fix Issues. This will clear the cache completely.
  • Preview No Download: Works on all devices, streaming only, no storage used. Safe for verifying Instagram and TikTok videos first.
  • Direct Download - Best for Android and Desktop: Uses an anchor tag with the download attribute. On Android Chrome, Firefox, Edge and on Windows, Mac, and Linux, it saves directly to the Downloads folder with your custom or formatted name.
  • Blob Save - Best for iPhone: iPhone Safari does not support the download attribute and opens the video instead. This method fetches the file as a blob, creates an object URL, and forces the save dialog.
  • File Picker Save - Modern Desktop Browsers: If your browser supports showSaveFilePicker, a system dialog will appear to let you choose the destination folder. If not supported, it falls back to Direct Download.
  • Open in New Tab: Fallback option. Opens the raw MP4 URL in a new tab. You can long press the video and select Save Video if other methods are blocked.
  • Share and Copy Link: Share opens the system share sheet on mobile devices, Copy places the direct link in your clipboard for use in Telegram or Notes.

We first attempt a HEAD request and read the Content-Length header. If the host blocks HEAD requests or CORS, we attempt a GET request with Range bytes 0-0 and read the total size from the Content-Range header, for example bytes 0-0 / 12345678. If the host blocks CORS headers, we retry the same requests through a proxy such as https://api.allorigins.win/raw and corsproxy.io which add Access-Control-Allow-Origin so the headers can be read. This is why the size can be displayed as 24.73 MB MP4 instead of unknown. If the host hides the size even through a proxy, we display Size hidden by host, use Preview to stream.
Nine detailed steps with timestamps. Step 1 is validation, Step 2 is engine check and wake, Step 3 is request building with quality and filename mode, Step 4 is POST to your Cobalt URL, Step 5 is waiting for 3 to 10 seconds, Step 6 is parsing and extracting URLs, Step 7 is filename generation, Step 8 is size detection, Step 9 is completion. If it stops at Step 4, the engine is in sleep mode. If an error occurs at Step 5, the link is private or unsupported.
  • Link Not Supported: The domain is not in the engine list. Please use a public link from Instagram, TikTok, YouTube, X, or Reddit.
  • Private or Unavailable: The post is private, deleted, or region locked. If it cannot be viewed in incognito mode, Pluck cannot access it either.
  • Age Restricted: Requires login for age verification. The engine does not bypass age restrictions by design.
  • Bad URL: Empty or invalid URL. The link must start with https:// and be copied from the Share button.
  • Engine Warming Up: Free hosting may enter sleep mode. Click Wake Engine, wait 8 to 10 seconds, and a green dot indicates readiness.
  • No File Returned: The content may be a livestream, DRM protected, or an expired story. Please try 720p.
  • Timeout: A network interruption occurred. Click Refresh Page to Fix Issues, which resolves the problem in most cases.
At the top of the page, we run a complete hard reset on every refresh or revisit. It clears localStorage, sessionStorage, cookies for this site, Cache API, IndexedDB, service workers, scroll restoration, and timeouts. Most issues are caused by outdated stored state. The refresh button performs the same hard reset and then reloads with a cache bust parameter _r equal to the current timestamp to ensure you receive a fresh copy.
